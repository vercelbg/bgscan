## مرور کلی

ابزار **bgscan** یک اسکنر شبکه چند پروتکله تحت ترمینال است که با زبان Go نوشته شده است. این ابزار آدرس‌های IP را در پروتکل‌های ICMP، TCP، HTTP، DNS و Xray بررسی می‌کند و مراحل اسکن را به صورت یک  (Pipeline) زنجیره‌ای اجرا می‌نماید؛ تمام این فرآیندها از طریق یک محیط متنی تعاملی (TUI) مبتنی بر کیبورد انجام می‌شود.

شما می‌توانید از این ابزار برای شناسایی هاست‌ها (Host Discovery)، اسکن پورت‌ها، تست وب‌سرویس‌ها، تحلیل رزولورهای DNS، شناسایی تونل‌ها و اعتبارسنجی پروکسی‌های Xray استفاده کنید. نتایج روی دیسک ذخیره می‌شوند و قابلیت اسکن مجدد دارند؛ بنابراین می‌توانید ابتدا یک اسکن وسیع با ICMP انجام دهید و سپس روی آی‌پی‌هایی که پاسخ داده‌اند، اسکن دقیق‌تری با TCP یا HTTP اجرا کنید.

این اسکنر برای توسعه‌دهندگان و پژوهشگرانی ساخته شده است که بدون ترک محیط ترمینال و کیبورد، به سرعت، انعطاف‌پذیری و یک تجربه مدرن در ترمینال نیاز دارند.

## قابلیت‌ها

**موتور اسکن**

- **بررسی چند پروتکله** — پشتیبانی از پروتکل‌های ICMP، TCP، HTTP/1.1، HTTP/2، HTTP/3 (QUIC)، TLS، DNS، DNSTT، Slipstream و Xray
- **زنجیره‌سازی اسکن ها** — اتصال مراحل مختلف اسکن به یکدیگر (به عنوان مثال ICMP ← TCP ← HTTP) با حالت‌های اجرای جریانی (Stream)، ترتیبی (Sequential) یا دسته‌ای (Batch)
- **ورکرهای همزمان** — امکان پیکربندی مخزن ورکرهای موازی (Worker Pools) به ازای هر پروب برای اسکن همزمان
- **بر زدن و نمونه‌برداری** — تصادفی‌سازی ترتیب اهداف و محدود کردن تعداد IPها برای اسکن

**رابط کاربری ترمینال (TUI)**

- **رابط کاربری BubbleTea TUI** — کاملاً مبتنی بر کیبورد؛ اسکن، مانیتورینگ و مدیریت نتایج بدون نیاز به مرورگر
- **داشبورد اسکن زنده** — نمودارهای پیشرفت تب‌بندی شده و جداول نتایج به ازای هر مرحله از اسکن
- **بازرس تنظیمات درون‌برنامه‌ای** — ویرایش تعاملی تمامی فیلدهای پیکربندی با قابلیت ذخیره‌سازی آنی روی دیسک
- **پشتیبانی از تم** — دارای پالت‌های رنگی تاریک، روشن و خودکار (بر پایه Catppuccin)

**داده‌ها و ورودی/خروجی (I/O)**

- **ذخیره و بازپخش** — نتایج در قالب فایل‌های CSV ذخیره می‌شوند و می‌توان از آن‌ها به عنوان ورودی اسکن‌های جدید استفاده کرد
- **لیست‌های IP همراه** — مجهز به رنج‌های آی‌پی سرویس‌های Cloudflare، AWS، Azure، Google، Akamai، Fastly، Bunny، G-Core و رنج‌های ایران
- **مدیریت بخش Outbound در Xray** — افزودن اوت‌باندها از طریق لینک‌های اشتراک‌گذاری یا فایل‌های JSON، به همراه امکان اعتبارسنجی و تست سرعت آن‌ها

**پایداری و قابلیت اطمینان**

- **نویسنده نتایج مقاوم در برابر کرش** — ادغام اتمیک داده‌ها با استفاده از `fsync` و تغییر نام فایل‌های موقت؛ بهینه‌سازی شده برای مصرف حافظه ثابت
- **چرخش لاگ‌ها (Log Rotation)** — ایجاد سه اسکن لاگ مجزا (هسته، رابط کاربری و دیباگ) با چرخش در حجم ۵۰ مگابایت، نگهداری ۳ نسخه پشتیبان و ماندگاری ۷ روزه
- **بررسی سلامت در زمان راه‌اندازی** — اعتبارسنجی تنظیمات، مکان‌یابی فایل‌های باینری اختیاری و نمایش هشدار به جای کرش کردن برنامه در صورت نبود برخی ابزارها

## چرا bgscan؟

- **یک ابزار، زنجیره کامل.** بیشتر اسکنرها فقط یک پروتکل را به خوبی پوشش می‌دهند. اما bgscan آن‌ها را به هم متصل می‌کند: یک رنج را پینگ کنید، به پورت‌های فعال متصل شوید و پورت‌های باز را با پروتکل HTTP بررسی کنید — همه این‌ها تنها در یک بار اجرا.
- **موتور مستقل از نوع اسکن.** موتور اصلی برنامه کاری به ماهیت پروب‌ها ندارد. این موتور آی‌پی‌ها را تحویل می‌گیرد، نتایج را جمع‌آوری کرده و روی دیسک می‌نویسد. افزودن یک نوع اسکن جدید صرفاً به معنای پیاده‌سازی یک اینترفیس `Probe` با سه متد و ثبت یک Stage Builder است.
- **بدون شلوغ‌کاری در زمان اجرا.** پروب‌های ICMP، TCP و HTTP از کتابخانه‌های استاندارد Go استفاده می‌کنند. باینری‌های Xray، DNSTT و Slipstream اختیاری هستند و در زمان راه‌اندازی اعتبارسنجی می‌شوند؛ در صورت نبود هر کدام، برنامه صرفاً یک هشدار ثبت کرده و تنها همان نوع اسکن را غیرفعال می‌کند.
- **پیکربندی فایل‌محور.** تمام تنظیمات در فایل‌های ساده TOML ذخیره می‌شوند. بازرس درون‌برنامه‌ای دقیقا همین فایل‌ها را می‌خواند و می‌نویسد؛ بنابراین هر چیزی که در رابط کاربری می‌بینید، عینا روی دیسک ذخیره شده است.
- **ساخته شده برای ترمینال.** ناوبری با کیبورد، دیالوگ‌های پاپ‌آپ (Overlay)، نمایش زنده پیشرفت، استریم لاگ‌ها — بدون نیاز به مرورگر، الکترون یا وب‌سرور.

## پروتکل‌های پشتیبانی شده

| پروتکل | لایه | توضیحات |
|:--------:|:-----:|-------------|
| **ICMP** | 3 | شناسایی هاست‌ها و بررسی دسترسی‌پذیری از طریق پینگ (Ping) |
| **TCP** | 4 | اسکن اتصالات و اعتبارسنجی دست‌تکانی TCP (TCP Handshake) |
| **HTTP** | 7 | بررسی پروتکل‌های HTTP/1.1، HTTP/2 و HTTP/3 (QUIC) از طریق ALPN |
| **TLS** | 7 | بررسی نسخه‌های TLS 1.0 تا TLS 1.3 |
| **DNS** | 7 | پرس‌وجوهای پیشرفته DNS (UDP, TCP, DNS-over-TLS) همراه با سیستم ضد دستکاری و جایگزینی (Anti-hijacking) |
| **DNSTT** | 7 | اعتبارسنجی تونل DNS (پروتکل SOCKS، بدون احراز هویت) |
| **Slipstream** | 7 | اعتبارسنجی تونل Slipstream (پروتکل SOCKS، بدون احراز هویت) |
| **Xray** | 7 | اعتبارسنجی خروجی‌های Xray (Outbounds) و تست پهنای باند و سرعت |

## نصب

### نصب سریع

**لینوکس / مک / ترموکس (Termux)**

```bash
curl -fsSL https://raw.githubusercontent.com/MohsenBg/bgscan/refs/heads/main/scripts/install.sh | bash
```

**ویندوز (PowerShell)**

```powershell
irm https://raw.githubusercontent.com/MohsenBg/bgscan/refs/heads/main/scripts/install.ps1 | iex
```

**اندروید (Termux)**

```bash
pkg update -y && pkg install bash curl unzip -y
curl -fsSL https://raw.githubusercontent.com/MohsenBg/bgscan/refs/heads/main/scripts/install.sh | bash
```

اسکریپت نصب‌کننده به طور خودکار پلتفرم شما را تشخیص داده، آخرین نسخه منتشر شده را دانلود کرده، آن را در پوشه `bgscan/` استخراج می‌کند و فایل باینری را قابل اجرا می‌سازد. در صورت اجرای مجدد، نصب قبلی را شناسایی کرده و پیشنهاد جایگزینی یا بک‌آپ‌گیری از آن را می‌دهد.

### نصب دستی

1. فایل ZIP متناسب با پلتفرم خود را از [صفحه نسخه‌ها (Releases)](https://github.com/MohsenBg/bgscan/releases/latest) دانلود کنید.
2. فایل فشرده را استخراج کنید.
3. فایل باینری را اجرا کنید:
   - **لینوکس / مک / ترموکس:** `./bgscan`
   - **ویندوز:** `bgscan.exe`

در اولین اجرا، یک پوشه به نام `settings/` حاوی تنظیمات پیش‌فرض TOML و یک پوشه به نام `ips/` شامل لیست‌های IP همراه ایجاد می‌شود.

### ساخت از روی سورس کد (Build)

ابزار bgscan از یک ابزار کمکی به نام `bgscan-builder` برای دریافت وابستگی‌های اختصاصی هر پلتفرم (باینری‌های Xray، DNSTT، Slipstream) و کامپایل پروژه استفاده می‌کند.

```bash
git clone https://github.com/MohsenBg/bgscan.git
cd bgscan

# نصب ابزار بیلدر
# لینوکس / مک:
curl -fsSL https://raw.githubusercontent.com/MohsenBg/bgscan/refs/heads/main/scripts/install-builder.sh | bash
# ویندوز (PowerShell):
irm https://raw.githubusercontent.com/MohsenBg/bgscan/refs/heads/main/scripts/install-builder.ps1 | iex

# دریافت وابستگی‌های پلتفرم شما
# لینوکس / مک:
./scripts/install-deps.sh
# ویندوز:
./scripts/install-deps.ps1

# کامپایل و اجرا
go run ./cmd/bgscan/
```

برای ساخت نسخه‌های نهایی (Release) برای یک پلتفرم خاص:

```bash
./bgscan-builder release -os linux -arch amd64
./bgscan-builder release -os windows -arch amd64
./bgscan-builder release -os darwin -arch arm64
```

> **نکته:** ابزار bgscan را نمی‌توان مستقیماً از طریق دستور `go install` نصب کرد، چرا که به باینری‌های خارجی اختصاصی پلتفرم (Xray, DNSTT, Slipstream) نیاز دارد. حتماً از ابزار بیلدر یا اسکریپت نصب سریع استفاده کنید.

## شروع سریع

1. برنامه bgscan را از پوشه نصب اجرا کنید (`./bgscan` در یونیکس، `bgscan.exe` در ویندوز).
2. گزینه **Run Scan** را انتخاب کرده و دکمه `Enter` را بزنید.
3. منبع هدف خود را مشخص کنید — **IP List** (از فایل‌های وارد شده شما) یا **Result List** (از نتایج اسکن‌های قبلی).
4. یک نوع اسکن انتخاب کنید — ICMP، TCP، HTTP، DNS یا Xray.
5. دکمه `Enter` را برای شروع فشار دهید. میزان پیشرفت و نتایج به صورت زنده در داشبورد استریم می‌شوند.
6. گزینه‌ی **Result Files** را از منوی اصلی باز کنید تا نتایج ذخیره شده را بررسی، تغییر نام یا حذف کنید.

| کلید | ناوبری و جابجایی |
|:---:|-------------|
| `↑` `↓` | جابجایی بین آیتم‌ها |
| `Enter` | انتخاب / شروع |
| `b` یا `Esc` | بازگشت به عقب |
| `q` | خروج از برنامه |

## تنظیمات (Configuration)

تمام تنظیمات در فایل‌های متنی ساده با فرمت TOML در پوشه `settings/` در کنار فایل باینری قرار دارند:

| فایل | کاربرد و هدف |
|------|---------|
| `general_settings.toml` | حالت جریان، حداکثر IPها، اندازه دسته‌ها (Batch Size)، بر زدن و بازه به‌روزرسانی وضعیت |
| `writer_settings.toml` | بافر کردن نتایج، بازه زمانی تخلیه بافر (Flush)، کانال و اندازه دسته‌ها |
| `icmp_settings.toml` | زمان انتظار (Timeout) در ICMP، تعداد تلاش مجدد و تعداد ورکرها |
| `tcp_settings.toml` | پورت TCP، زمان انتظار، تعداد تلاش مجدد و تعداد ورکرها |
| `http_settings.toml` | نسخه HTTP/HTTPS/HTTP3، رنج TLS، کدهای وضعیت (Status Codes) قابل قبول |
| `dns_settings.toml` | تنظیمات رزولور DNS، بهینه‌سازی‌های DNSTT و Slipstream |
| `xray_settings.toml` | نوع تست اتصال Xray، تست سرعت و فرآیند پیش‌اسکن (Pre-scan) |

شما می‌توانید فایل‌ها را به صورت دستی ویرایش کنید یا از بازرس درون‌برنامه‌ای استفاده کنید — هر دو روش روی فایل‌های یکسانی می‌نویسند. بازرس تغییرات را بدون نیاز به ریستارت برنامه، به صورت آنی روی دیسک ذخیره می‌کند. برای دیدن جزئیات هر فیلد به [مستندات تنظیمات](https://mohsenbg.github.io/bgscan/docs/settings/) مراجعه کنید.

## نمونه‌های کاربردی

**اسکن یک لیست IP همراه**

پس از اجرای برنامه، مسیر **Run Scan → IP List → cloudflare_IPv4** را انتخاب کنید، سپس یک نوع اسکن را برگزینید (مثلا کلید `t` برای TCP). ابزار bgscan لیست را با ورکرهای تنظیم‌شده اسکن کرده و نتایج را در پوشه `result/tcp/` ذخیره می‌کند.

**زنجیره‌سازی  ICMP → TCP → HTTP**

در فایل `general_settings.toml` مقدار فیلد `pipeline_mode` را برابر با `"streaming"` قرار دهید. اگر چندین نوع اسکن فعال باشد، مراحل آن‌ها به طور خودکار به هم زنجیر می‌شوند — به این صورت که تنها IPهایی که معیارهای موفقیت مرحله قبلی را پاس کنند به مرحله بعدی فرستاده می‌شوند.

**اسکن مجدد نتایج قبلی**

مسیر **Run Scan → Result List** را انتخاب کرده و یکی از فایل‌های نتایج ذخیره‌شده را برگزینید. ابزار bgscan تنها آی‌پی‌های موجود در آن فایل را مجدداً اسکن می‌کند؛ این کار برای تحلیل عمیق‌تر روی هاست‌هایی که از مراحل اولیه عبور کرده‌اند بسیار کاربردی است.

**اعتبارسنجی اوت‌باندهای Xray**

از منوی اصلی، بخش **Xray → Outbounds** را باز کنید، کلید `a` را فشار دهید تا یک قالب (Template) از طریق لینک اشتراک‌گذاری (مانند `vless://`, `vmess://`, `trojan://`, `ss://`, `hysteria2://`, `wireguard://`) یا فایل JSON اضافه کنید. سپس یک اسکن Xray اجرا کنید تا وضعیت اتصال و پهنای باند آن‌ها را بسنجید.

```
bgscan/
├── cmd/bgscan/              # نقطه ورود برنامه (Entry Point)
├── internal/
│   ├── core/
│   │   ├── config/          # تنظیمات TOML + اعتبارسنج‌ها
│   │   ├── scanner/         # هماهنگ‌کننده اسکنر، موتور اصلی، پروب‌ها و مدیریت پورت
│   │   ├── result/          # نویسنده ناهمگام (Async)، ادغام CSV، ثبت و بارگذاری نتایج
│   │   ├── iplist/          # بارگذاری، پارس کردن، ثبت و بر زدن لیست‌های IP
│   │   ├── dns/             # ابزارهای کمکی پرس‌وجوی DNS، ابزارهای DNSTT، Slipstream و SOCKS5
│   │   ├── xray/            # اجراکننده Xray، پارس کردن لینک‌ها/اوت‌باندها و تست سرعت
│   │   ├── process/         # مدیریت چرخه حیات پروسه‌ها به صورت مالتی‌پلتفرم
│   │   └── fileutil/        # ابزارهای کمکی برای کار با فایل‌های CSV، JSON، TOML، متنی و موقت
│   ├── logger/              # سیستم ثبت لاگ سطح‌بندی شده همراه با چرخش خودکار Lumberjack
│   ├── startup/             # بررسی‌های سلامت اولیه (لاگر، کانفیگ، xray، dnstt، slipstream)
│   └── ui/                  # رابط کاربری BubbleTea TUI (کامپوننت‌ها، منوها، جداول و تم)
├── assets/                  # باینری‌های Xray، DNSTT، Slipstream + قالب‌های اوت‌باند
├── ips/                     # لیست‌های IP همراه و وارد شده (فرمت CSV)
├── settings/                # فایل‌های پیکربندی پیش‌فرض TOML
├── result/                  # خروجی‌های اسکن (CSV، تفکیک شده بر اساس نوع اسکن)
├── logs/                    # فایل‌های هسته (core.log)، رابط کاربری (ui.log) و دیباگ (debug.log)
├── scripts/                 # اسکریپت‌های نصب، ساخت، مدیریت وابستگی‌ها و انتشار نسخه‌ها
├── docs/                    # سایت مستندات ساخته شده با Hugo Book
├── go.mod
```

## مشارکت در پروژه (Contributing)

مشارکت‌های شما — شامل رفع باگ‌ها، بهبود مستندات و افزودن قابلیت‌های جدید — با آغوش باز پذیرفته می‌شود.

- **از شاخه `main` انشعاب (Branch) بگیرید** و از یک نام توصیفی برای شاخه خود استفاده کنید (مانند `feature/`, `fix/`, `docs/`, `refactor/`).
- **از کنوانسیون‌های موجود پیروی کنید** — کد را ساده و خوانا نگه دارید و ترجیحاً کامیت‌های کوچک و متمرکز بزنید.
- **پیام‌های کامیت** — از پیشوندهای استاندارد مانند `feat:`، `fix:`، `docs:`، `refactor:` و `test:` استفاده کنید.
- **قبل از باز کردن PR (درخواست ادغام)** — مطمئن شوید پروژه بدون خطا کامپل می‌شود، تست‌های موجود با موفقیت پاس می‌شوند و مستندات برای قابلیت‌های جدید به‌روزرسانی شده‌اند.
- **برای تغییرات بزرگ** — ابتدا یک Issue باز کنید تا در مورد رویکرد و نحوه پیاده‌سازی آن گفتگو کنیم.

برای اطلاعات بیشتر، راهنمای کامل [مشارکت در پروژه](https://mohsenbg.github.io/bgscan/docs/developer/contributing/) را در مستندات مطالعه کنید.
