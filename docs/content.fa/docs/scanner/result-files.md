# فایل‌های نتایج (Result Files)

پس از اتمام هر اسکن، bgscan آی‌پ‌ی‌های کشف‌شده و معیارهای سنجش آن‌ها را در فایل‌های نتایج می‌نویسد. این فایل‌ها بر اساس زمان ساخت سازماندهی می‌شوند و می‌توانید از طریق محیط TUI آن‌ها را مرور، مشاهده، تغییر نام یا حذف کنید.

---

## مشاهده فایل‌های نتایج

برای باز کردن مرورگر فایل‌های نتایج، به مسیر **Main Menu → Result Files** بروید.

در جدول این صفحه، تمام فایل‌های نتایج مربوط به انواع اسکن‌ها به ترتیب از جدیدترین به قدیمی‌ترین مرتب شده‌اند و اطلاعات زیر را نشان می‌دهند:

| ستون | توضیحات |
|---|---|
| File Name | نام فایل نتیجه (بدون پسوند `.csv`) |
| Created Time | برچسب زمانی ساخت فایل یا آخرین تغییر آن |
| Type | موتور اسکنی که این فایل را تولید کرده است (`icmp`، `tcp`، `http` و...) |
| Size | حجم فایل روی دیسک |

دو اکشن با کیبورد در این صفحه در دسترس است:

| کلید | اکشن |
|---|---|
| `r` | تغییر نام فایل انتخاب‌شده |
| `x` | حذف دائمی فایل انتخاب‌شده |

#### باز کردن یک فایل نتیجه

روی هر کدام از فایل‌ها که کلید Enter را بزنید، بخش مشاهده آی‌پی (IP viewer) باز می‌شود. در این بخش، آی‌پی‌های پیداشده در آن فایل به همراه مقادیر تأخیر (Latency)، سرعت دانلود و سرعت آپلود نمایش داده می‌شوند. نتایج اسکن Xray معیارهای کامل دانلود و آپلود را نشان می‌دهند؛ در حالی که بقیه انواع اسکن‌ها یک نمای خلاصه شده که فقط شامل تأخیر است را به شما نمایش خواهند داد.

## محل ذخیره‌سازی

فایل‌های نتایج در پوشه `result/` در کنار فایل اجرایی bgscan ذخیره می‌شوند. هر نوع اسکن، پوشه اختصاصی خودش را دارد:

```text
/
└── result/
├── icmp/
├── tcp/
├── http/
├── xray/
├── resolve/
├── dnstt/
└── slipstream/

```

هر پوشه حاوی فایل‌های `.csv` تولید شده توسط موتور اسکن مربوط به خود است. این پوشه‌ها هنگام اجرای اسکن به‌طور خودکار ساخته می‌شوند و نیازی نیست آن‌ها را به صورت دستی ایجاد کنید.

## نام‌گذاری فایل‌ها

فایل‌های نتایج بر اساس پیشوندِ تنظیم‌شده برای هر موتور اسکن و یک برچسب زمانی نام‌گذاری می‌شوند:

```text
<YYYYMMDD_HHMMSS>.csv

```

هر نوع اسکن یک پیشوند پیش‌فرض دارد که در تنظیمات آن مشخص شده است:

| نوع اسکن | پیشوند پیش‌فرض | نمونه نام فایل |
|---|---|---|
| ICMP | `icmp_` | `icmp_20260711_143022.csv` |
| TCP | `tcp_` | `tcp_20260711_143022.csv` |
| HTTP | `http_` | `http_20260711_143022.csv` |
| Xray | `xray_` | `xray_20260711_143022.csv` |
| DNS Resolver | `dns_resolver_` | `dns_resolver_20260711_143022.csv` |
| DNSTT | `dns_dnstt_` | `dns_dnstt_20260711_143022.csv` |
| SlipStream | `dns_slipstream_` | `dns_slipstream_20260711_143022.csv` |

شما می‌توانید پیشوند هر نوع اسکن را در فایل تنظیمات مربوط به آن در پوشه `settings/` تغییر دهید. برای جزئیات بیشتر، بخش [پیکربندی (Configuration)](../configuration.md) را بررسی کنید.

## فرمت فایل‌ها

فایل‌های نتایج به صورت CSV ساده و بدون ردیف هدر (سرفصل) هستند. هر ردیف نشان‌دهنده یک آی‌پی پاسخگو است:

| فیلد | توضیحات |
|---|---|
| `ip` | آدرس IPv4 که پاسخ داده است |
| `latency` | تأخیر رفت و برگشت یا زمان اتصال (مثلاً `123ms`) |
| `download` | مقدار سنجش دانلود — اگر اندازه‌گیری نشده باشد `0s` است |
| `upload` | مقدار سنجش آپلود — اگر اندازه‌گیری نشده باشد `0s` است |

#### نمونه

```csv
1.2.3.4,45ms,0s,0s
5.6.7.8,120ms,320ms,80ms

```

در اسکن‌هایی که سرعت را اندازه‌گیری نمی‌کنند (مانند ICMP، TCP، HTTP، DNS)، ستون‌های دانلود و آپلود همچنان وجود دارند اما مقدار آن‌ها روی `0s` تنظیم می‌شود.

## رتبه‌بندی و ترتیب نتایج

آی‌پی‌های داخل یک فایل نتیجه بر اساس یک امتیاز کیفی مرتب می‌شوند که هر چه این امتیاز بالاتر باشد، کیفیت آی‌پی بهتر است. این امتیاز از ترکیب هر سه معیار زیر محاسبه می‌شود:

* **۶۰٪** سرعت دانلود
* **۲۰٪** سرعت آپلود
* **۲۰٪** میزان تأخیر (Latency)

این یعنی بهترین آی‌پی‌ها (سریع‌ترین دانلود و کمترین تأخیر) در بالای فایل قرار می‌گیرند. اگر دو آی‌پی امتیاز کاملاً یکسانی داشته باشند، حروف الفبای آدرس آی‌پی ملاک رتبه‌بندی قرار می‌گیرد تا اولویت مشخص شود.
