# انواع اسکن (Scan Types)

ابزار bgscan از ۵ نوع اسکن اصلی پشتیبانی می‌کند که از طریق منوی اصلی و با کلیدهای میانبر کیبورد قابل دسترسی هستند:

| کلید | نوع اسکن | توضیحات | کاربرد اصلی | فایل پیکربندی |
|-----|-----------|-------------|------------------|-------------------|
| `i` | **اسکن ICMP** | ارسال درخواست‌های ICMP echo (پینگ) برای بررسی در دسترس بودن هاست | کشف هاست‌های فعال، نقشه‌برداری شبکه | [`icmp_settings.toml`](../settings/icmp.md) |
| `t` | **اسکن TCP** | تلاش برای برقراری اتصال TCP به پورت‌های مشخص شده | اسکن پورت‌ها، شناسایی سرویس‌ها | [`tcp_settings.toml`](../settings/tcp.md) |
| `h` | **اسکن HTTP** | ارسال درخواست‌های HTTP/HTTPS به وب‌سرورها | تست وب‌سرویس‌ها، کشف اندپوینت‌های API | [`http_settings.toml`](../settings/http.md) |
| `d` | **اسکن DNS** | پرس‌وجو از سرورهای DNS برای بررسی قابلیت‌های رزولوشن و تانلینگ | تست ریزالورهای DNS، شناسایی قابلیت تانلینگ | [`dns_settings.toml`](../settings/dns.md) |
| `x` | **اسکن Xray** | تست اتصال خروجی (Outbound) و سنجش پهنای باند به سرورهای مقصد | تست خروجی شبکه، اندازه‌گیری پهنای باند | [`xray_settings.toml`](../settings/xray.md) |

> 💡 **نکته**: گزینه اسکن DNS) `d`)  در صورت فعال بودن در تنظیمات، خودش شامل سه نوع زیرمجموعه اسکن می‌شود:
>
> - **DNS Resolve**: پرس‌وجوهای استاندارد DNS (رکوردها از نوع A/AAAA/TXT)
> - **DNSTT**: تست ابزار تانل DNS (فناوری DNSTT) برای بررسی قابلیت تانلینگ
> - **SlipStream**: روشی جایگزین برای شناسایی تانل‌های DNS
>
> این زیرمجموعه‌ها به صورت مجزا در فایل تنظیمات DNS کنترل و مدیریت می‌شوند.

## جزئیات مربوط به انواع اسکن

#### اسکن ICMP

- **پروتکل**: ICMP Echo Request/Reply (همان پینگ خودمان)
- **کارکرد**: درخواست‌های اکو را به آی‌پی‌های هدف می‌فرستد و زمان رفت و برگشت پاسخ‌ها (RTT) را اندازه‌گیری می‌کند.
- **خروجی**: لیستی از هاست‌های پاسخگو به همراه زمان پاسخ‌دهی آن‌ها.
- **تنظیمات کلیدی**: مهلت زمانی (Timeout)، تعداد تلاش مجدد (Retries)، تعداد کانکارنسی ورکرها (Worker concurrency).
- **بهترین کاربرد**: شناسایی اولیه هاست‌های فعال، قبل از اجرای اسکن‌های سنگین‌تر.

#### اسکن TCP

- **پروتکل**: اسکن از نوع TCP SYN/connect
- **کارکرد**: تلاش می‌کند تا اتصال‌های TCP روی پورت‌های مشخص شده با آی‌پی‌های هدف برقرار کند.
- **خروجی**: لیستی از هاست‌هایی که پورت باز دارند به همراه زمان برقراری اتصال.
- **تنظیمات کلیدی**: پورت هدف، مهلت زمانی، تعداد تلاش مجدد، تعداد کانکارنسی ورکرها.
- **بهترین کاربرد**: کشف سرویس‌ها، اسکن پورت‌ها و تست فایروال.

#### اسکن HTTP

- **پروتکل**: HTTP/HTTPS (پشتیبانی از HTTP/1.1، HTTP/2 و HTTP/3 از طریق ALPN)
- **کارکرد**: درخواست‌های HTTP را به هاست‌ها و پورت‌های هدف می‌فرستد و پاسخ‌ها را ارزیابی می‌کند.
- **خروجی**: لیستی از وب‌سرورهای پاسخگو به همراه استاتوس کدها، زمان پاسخ‌دهی و هدرها/محتوای اختیاری.
- **تنظیمات کلیدی**: هاست/پورت هدف، پروتکل (HTTP/HTTPS)، اعتبارسنجی TLS، نسخه HTTP، استاتوس کدهای مجاز.
- **بهترین کاربرد**: تست وب‌سرویس‌ها، کشف اندپوینت‌های API و بررسی لود بالانسرها.

#### اسکن DNS

- **پروتکل**: پرس‌وجوهای DNS (از طریق UDP، TCP یا DNS-over-TLS)
- **کارکرد**: درخواست‌های DNS را به ریزالورهای هدف می‌فرستد و پاسخ‌های آن‌ها را ارزیابی می‌کند.
- **خروجی**: لیستی از ریزالورهای DNS فعال به همراه زمان پاسخ‌دهی و قابلیت‌های آن‌ها.
- **تنظیمات کلیدی**: ورکرهای ریزالور، پروتکل، دامین، انواع کوئری، مهلت زمانی، تلاش مجدد، کدهای پاسخ مجاز، بررسی‌های ضد هجکینگ (Anti-hijacking).
- **بهترین کاربرد**: پیدا کردن دی‌ان‌اس سرورهای باز و پاسخگو، تست تنظیمات امنیتی DNS.

#### اسکن Xray

- **پروتکل**: پروتکل اختصاصی Xray برای تست اتصال و پهنای باند
- **کارکرد**: اتصال شبکه خروجی را بررسی کرده و سرعت آپلود/دانلود را به سرورهای کانفیگ‌شده اندازه‌گیری می‌کند.
- **خروجی**: لیستی از سرورهای در دسترس به همراه زمان اتصال و پهنای باند اندازه‌گیری شده.
- **تنظیمات کلیدی**: مهلت زمانی، تعداد ورکرها، نوع تست (فقط اتصال/دانلود/آپلود/هر دو)، حجم داده برای تست‌های انتقال فایل.
- **بهترین کاربرد**: تست کیفیت خروجی شبکه، اندازه‌گیری سرعت و پهنای باند، شناسایی پروکسی‌ها و گلوگاه‌های شبکه.
