## ICMP Settings

These settings control the behavior of the ICMP scanner, including response timeouts, retry attempts, concurrency levels, and output file naming.

---

### `timeout`

```toml
timeout = 2000
```

Maximum time to wait for an ICMP reply from a target, measured in **milliseconds**.

If no response is received within this period, the target is considered **unreachable**.

| Value  | Behavior                              |
| ------ | ------------------------------------- |
| `1000` | Wait up to 1 second                   |
| `2000` | Wait up to 2 seconds (default)        |
| `5000` | Suitable for slow or distant networks |

> Increasing this value may improve detection rates on high-latency networks, but it also increases total scan time.

---

### `tries`

```toml
tries = 1
```

Number of ICMP probe attempts sent to each target.

If a probe does not receive a response, the scanner will retry until the configured limit is reached.

| Value | Behavior                                |
| ----- | --------------------------------------- |
| `1`   | Single attempt (default)                |
| `2`   | One retry if no response is received    |
| `3+`  | Higher reliability on unstable networks |

> Higher values can improve accuracy but will increase scan duration.

---

### `workers`

```toml
workers = 1000
```

Number of concurrent worker goroutines responsible for sending ICMP probes.

Higher values increase scanning speed but also increase CPU usage, memory consumption, and network load.

| Value   | Behavior                                                    |
| ------- | ----------------------------------------------------------- |
| `100`   | Suitable for low-resource systems                           |
| `500+`  | Balanced performance and resource usage                     |
| `1000+` | Suitable for high-performance servers and large-scale scans |

> The optimal value depends on available system resources and network capacity.

---

### `prefix_output`

```toml
prefix_output = "icmp_"
```

Prefix used when naming files generated by this scanner.

This is useful when multiple scanner types are executed within the same project and their outputs need to be distinguished.

| Value       | Example Output       |
| ----------- | -------------------- |
| `"icmp_"`   | `icmp_results.txt`   |
| `"ping_"`   | `ping_results.txt`   |
| `"custom_"` | `custom_results.txt` |

> This setting only affects output file names and does not change scanning behavior.
